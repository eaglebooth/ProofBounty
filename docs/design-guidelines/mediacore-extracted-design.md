# MediaCore-Inspired Design Guidelines

Reference screenshots from `https://mediacore.framer.ai/`.

## Visual Direction

- Bright editorial SaaS page with a mostly white background.
- Hero content is centered, airy, and confident, with a soft lime glow band near the lower hero.
- Black typography is heavy and rounded; script accent words add contrast.
- CTA color is neon lime (`#6dff18` / close variants), with black text and rounded pill geometry.
- Secondary buttons are white with thin gray borders.
- Orange-red bullet accents are used sparingly in service sections.

## Layout Patterns

- Header: logo left, simple nav centered, bright CTA right.
- Hero: pill trust badge, huge centered headline, short description, primary CTA.
- Service sections: two-column alternating layout with large rounded images or product mock panels.
- Process section: left headline + right vertical timeline with lime active node.
- Rounded corners are generous for media panels, around 24px; input/tool cards can use 18-24px.

## ProofBounty Adaptation

- Keep the bright MediaCore look, but make the first screen useful for a GitHub bounty marketplace.
- Replace passive marketing sections with app actions:
  - Create bounty
  - Submit GitHub PR/issue/commit URL
  - AI evaluate contribution
  - Release reward
- Use a product visual mock instead of generic agency photography: a large GitHub PR review board and bounty ledger.
- Keep all app copy in English for submission.
